>> Introduction


This control enables you to create a toolbar.


>> Control Styles


These control styles are available when creating a toolbar control.

Control Styles
nodivider   The 2 pixel line divider is not included in the toolbar.
flat   Toolbar style is flat with buttons having a hot mode (hover). This style also includes the transparent style.
list   Toolbar item text is to the right of the icon.
transparent   Toolbar control is transparent letting the dialog background color through.
tooltips   Toolbar items support tooltips.
wrap   Toolbar can have multiple lines of buttons. Toolbar buttons "wrap" to the next line when the toolbar becomes too narrow to include all buttons on the same line. The break will occure on the rightmost separator item or the rightmost button if there are no separators on the bar.
arrows   Displays a dropdown arrow on the buttons having the +v flag set.
top   The toolbar is auto-positioned at top of dialog window.
bottom   The toolbar is auto-positioned at bottom of dialog window.
left   Toolbar is auto-positioned to the left side of the dialog (gives a vertical toolbar).
right   Toolbar is auto-positioned to the right side of the dialog (gives a vertical toolbar).
noauto   Prevents the toolbar from being auto-positioned and resized by the parent window. The user is responsible for the positioning and resizing of the toolbar.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add an item to the toolbar.

Command: /xdid -a dialog ID [N] [+FLAGS] [WIDTH] [#ICON] [COLOR] [Button Text][TAB]Tooltip Text

Example: /xdid -a dcx 4 1 +cv 100 1 $rgb(255,0,0) Button $chr(9) Tooltip

Parms :
[N]Button index number. (Use 0 to insert at the end)
[+FLAGS]Item flags
  Value :
aThe button is auto-sized to fit in the icon and text.
bThe button text is bold. (Only works with XP themes disabled)
cThe button text is colored ([COLOR] is the text color). (Only works with XP themes disabled)
dThe button is disabled.
gThe button is part of a checkgroup (radio checks) (used with -k flag).
hThe button is hidden (but still part of the toolbar).
iThe button is in indeterminate state (greyed out).
kThe button acts like a checkbox.
lSpecifies that [WIDTH] is the button width.
pThe button appears like pressed.
uItem text is underlined. (Only works with XP themes disabled)
vThe button acts like a dropdown button. (If the arrows style is specified, a dropdown arrow will be drawn beside the button)
wThe button next to a button having this style will be forced to the next line (and all other buttons following).
xThe button is checked.

[WIDTH]Button width (Use 0 if not used).
[#ICON]Icon displayed when item is not selected.
[COLOR]Toolbar button text color (Only works with XP themes disabled).

N.B. Button text is optional.

N.B. If Button Text is '-' the button will be a separator item.

N.B. If the button is a separator item, the value used for ICON will be the separator width in pixels.

N.B. Icons must be added prior to be used in the toolbar. (see -l)

• -c


This command lets you change the Nth toolbar button color and text styles.

Command: /xdid -c dialog ID [N] [+FLAGS] [COLOR]

Example: /xdid -c dcx 4 3 +bc $rgb(0,0,255)

N.B. The flags are limited to +bcu.

• -d


This command lets you delete the Nth toolbar button.

Command: /xdid -d dialog ID N

Example: /xdid -d dcx 4 6

• -i


This command lets you Nth toolbar button icon (Use 0 for no icon).

Command: /xdid -i dialog ID [N] [ICON]

Example: /xdid -i dcx 4 7 2

• -j


This command lets you set the minimum and maximum width of toolbar buttons.

Command: /xdid -j dialog ID [MIN] [MAX]

Example: /xdid -j dcx 4 50 100

• -l


This command lets you set the icon size of the toolbar image lists.

Command: /xdid -l dialog ID [SIZE]

Example: /xdid -l dcx 4 16

Parms :
[SIZE]Image list image size
  Value :
16Icons are 16x16 pixels.
24Icons are 24x24 pixels.
32Icons are 32x32 pixels.
N.B.This command can also be used to clear the images lists.

N.B.This command must be issued before adding icons to the toolbar image lists.

• -m


This command lets you toggle the toolbar autostretch feature. This feature resizes automatically the toolbar buttons to stretch the full width of the toolbar when the control width changes. Does not work on vertical toolbars.

Command: /xdid -m dialog ID [1|0]

Example: /xdid -m dcx 4 1

• -q


This command lets you change the Nth button tooltip text (text is optional).

Command: /xdid -q dialog ID [N] (TEXT)

Example: /xdid -q dcx 4 2 New TipText

• -r


This command lets you clear all the toolbar items.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -t


This command lets you change the state of a toolbar button.

Command: /xdid -t dialog ID [N] [+FLAGS]

Example: /xdid -t dcx 4 2 +d

Parms :
[N]Button index number
[+FLAGS]Item flags
  Value :
dThe button is disabled.
hThe button is hidden (but still part of the toolbar).
iThe button is in indeterminate state (greyed out).
pThe button appears like pressed.
wThe button next to a button having this style will be forced to the next line (and all other buttons following).
xThe button is checked.

N.B. A flags value of + resets the button to it's normal state.

• -u


This command lets change the toolbar default button size.

Command: /xdid -u dialog ID [WIDTH] [HEIGHT]

Example: /xdid -u dcx 4 20 15

• -v


This command lets you change the Nth button text (text is optional).

Command: /xdid -v dialog ID [N] (TEXT)

Example: /xdid -v dcx 4 2 New Text

• -w


This command lets you add an icon to the toolbar image lists.

Command: /xdid -w dialog ID [+FLAGS] [INDEX] [FILENAME]

Example: /xdid -w dcx 4 +n 113 C:/mIRC/shell.dll

Parms :
[+FLAGS]Image list flags
  Value :
dDisabled icon list.
nNormal icon list.
hHottrack icon list.

[INDEX]Icon index in icon archive (use 0 if the file is a single icon file)
[FILENAME]Icon archive filename


>> $xdid() Properties


The following properties are used to retreive information in the toolbar control.

• .num


This property lets you retreive the total number of buttons on the toolbar.

Identifier: $xdid(dialog, ID).num

Example: //echo -a $xdid(dcx, 4).num

• .text


This property lets you retreive the Nth button text.

Identifier: $xdid(dialog, ID, N).text

Example: //echo -a $xdid(dcx, 4, 1).text

• .state


This property lets you retreive the state of the Nth toolbar button. The flags returned are in the same format specified here.

Identifier: $xdid(dialog, ID, N).state

Example: //echo -a $xdid(dcx, 4, 6).state

• .icon


This property lets you retreive the Nth button icon.

Identifier: $xdid(dialog, ID, N).icon

Example: //echo -a $xdid(dcx, 4, 1).icon

• tooltip


This property lets you retreive the Nth toolbar button tooltip.

Identifier: $xdid(dialog, ID, N).tooltip

Example: //echo -a $xdid(dcx, 4, 1).tooltip


>> Control Events


The following events are fired when a user does some particular actions on the toolbar control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   When a button is clicked.
 
rclick   When a the right mouse button is clicked over a button.
 
[VALUE][Button ID] [X_Bottom] [Y_Bottom] [X_Top] [Y_Top]
[Button ID]Button that triggered the event
[X_Bottom] [Y_Bottom]Coordinates of a possible popup menu aligned under the button where the rclick was fired.
[X_Top] [Y_Top]Coordinates of a possible popup menu aligned over the button where the rclick was fired.
dropdown   When a dropdown button (or arrow if arrows style is set) event is fired on a button.
 
[VALUE][Button ID] [X_Bottom] [Y_Bottom] [X_Top] [Y_Top]
[Button ID]Button that triggered the event.
[X_Bottom] [Y_Bottom]Coordinates of a possible popup menu aligned under the button where the dropdown was fired.
[X_Top] [Y_Top]Coordinates of a possible popup menu aligned over the button where the dropdown was fired.
 
help   Launched when you click on a control using the ? contexthelp button.
