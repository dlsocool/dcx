>> Control Styles


These control styles are available when creating a lisbox control.

Control Styles
noscroll   Shows a disabled vertical scroll bar for the list box when the box does not contain enough items to scroll. If you do not specify this style, the scroll bar is hidden when the list box does not contain enough items.
extsel   Allows multiple items to be selected by using the SHIFT key and the mouse or special key combinations.
multi   Turns string selection on or off each time the user clicks or double-clicks a string in the list box. The user can select any number of strings.
nointegral   Specifies that the size of the list box is exactly the size specified by the application when it created the list box. Normally, the system sizes a list box so that the list box does not display partial items.
nosel   Specifies that the list box contains items that can be viewed but not selected.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add items to the listbox.

Command: /xdid -a dialog ID [N] [Item Text]

Example: /xdid -a dcx 4 0 Item 1

N.B. You can use the value 0 for N to insert at the end of the listbox.

• -c


This command lets you check items in the listbox.

Command: /xdid -c dialog ID [N,[N,[...]]]

Example: /xdid -c dcx 4 1,2,3

N.B. In a single line listbox, you only provide the line number to select.

• -d


This command lets you delete an item in the listbox.

Command: /xdid -d dialog ID [N]

Example: /xdid -d dcx 4 1

• -r


This command lets you clear the listbox contents.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -u


This command lets you uncheck the listbox selection.

Command: /xdid -u dialog ID

Example: /xdid -u dcx 4

>> $xdid() Properties


The following properties are used to retreive information in the listbox control.

• text


This property lets you retreive the Nth listbox item text.

Identifier: $xdid(dialog, ID, N).text

Example: //echo -a $xdid(dcx, 4, N).text

• .num


This property lets you retreive the number of listbox items.

Identifier: $xdid(dialog, ID).num

Example: //echo -a $xdid(dcx, 4).num

• .sel


This property lets you retreive the selected item(s).

Identifier: $xdid(dialog, ID).sel

Example: //echo -a $xdid(dcx, 4).sel

N.B. In a multiselection listbox, it returns a comma separated list of selected indexes.

>> Control Events


The following events are fired when a user does some particular actions on the check control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   When an item is selected in the listbox.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
 
dclick   When an item is double-clicked in the listbox.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
 
rclick   When you right-click on the list.
 
help   Launched when you click on a control using the ? contexthelp button.
