>> Introduction


This control enables you to create a listview in 4 different view styles: icon, small icon, list and report.


>> Control Styles


These control styles are available when creating a listview control. Some of the styles must be used exclusively (report, icon, smallicon, list). Others can only be used when having a particular listview view (ex: grid only works with report view).

Control Styles
icon   Creates a listview in icon view
smallicon   Creates a listview in smallicon view
list   Creates a listview in item list view
report   Creates a listview in report view (the view with columns)
 
singlesel   Allows only one item at a time to be selected. By default, multiple items can be selected.
flatsb   Enables Flat Scroll Bars on the listview.
borderselect   When an item is selected a special border is drawn arround it instead of it being highlighted.
grid   Displays grid lines between item cells. (report view) (gridlines don't work with groups)
fullrow   Highlights the full row when an item is selected. (report view)
noheader   Does not display the header on the listview. (report view)
alignleft   Items are left-aligned in the listview control. (icon & smallicon views)
aligntop   Items are aligned with the top of the listview control. (icon & smallicon views)
autoarrange   Items are re-arranged automaticly when inserted. (icon & smallicon views)
showsel   When the control loses focus, the selection is still displayed.
nolabelwrap   The text is displayed on a single line. (icon & smallicon views)
checkbox   Enables Checkbox support in the list view. (report view)
editlabel   Enables the possibility to edit listview item labels.
headerdrag   Enables the possibility to drag listview headers to another position.
sortasc   Items are sorted based on item text in ascending order. (Only works with report or list)
sortdesc   Items are sorted based on item text in descending order. (Only works with report or list)
subitemimage   Subitems have images. (report view)
noscroll   Scrolling is disabled. All items must be within the client area. (icon & smallicon views)
hottrack   Enables hot-track selection in a listview control. Hot track selection means that an item is automatically selected when the cursor remains over the item for a certain period of time.
tooltip   If a partially hidden label in any view lacks tooltip text, the control will unfold the label. If this style is not set, the control will unfold partly hidden labels only for the icon view.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add data to the listview control.

Command: /xdid -a dialog ID [N] [INDENT] [+FLAGS] [#ICON] [#STATE] [#OVERLAY] [#GROUPID] [COLOR] [BGCOLOR] Item Text[TAB][+FLAGS] [#ICON] Item Text[TAB]...

Example: /xdid -a dcx 5 1 0 +cbku 1 0 0 0 -1 $rgb(170,213,255) text $chr(9) + 0 text2 $chr(9) + 0 text3

Parms :
[N]Position where the listview item is inserted. (Use 0 to insert at the end)
[INDENT]Space in icon width at the beginning of a line in the listview (0 for no space)
[+FLAGS]Item flags
  Value :
bThe item is bold.
cThe item has a text color defined by the [COLOR] parameter (use -1 for no color).
dThe item appears selected like a drop target.
fThe item has focus thus having a dotted box around it.
kThe item has a background color defined by the [BGCOLOR] parameter (use -1 for no color).
sThe item is selected.
tThe item appears as 50% opaque, looks like it has a "ghost" effect.
uThe item is underlined.

[#ICON]Icon index number from the normal icon list (0 for no icon)
[#STATE]Icon index number from the state icon list (0 for no icon) (If you are using the checkbox style -> 2: checked, 1: not checked, 0: no checkbox)
[#OVERLAY]Icon index number from the normal icon list that was marked as an overlay icon (0 for no icon) (Not functionnal for the moment)
[#GROUPID]Group ID (XP+) (Use 0 for no value)
[COLOR]Item text color.
[BGCOLOR]Item background color.

N.B. Icons must be added prior to be used in the listview.
N.B. The [INDENT] parameter only works if you are using normal icons in the listview.
N.B. If you add icons to the icon lists and use 0 for no icon, there will be the an icon indent space in front of the item text. (normal and state lists)

• -c


This command lets you select listview items.

Command: /xdid -c dialog ID [N,[N,[..]]]

Example: /xdid -c dcx 4 1,3,5

N.B. In a singlesel listview there is only one value for N.

• -d


This command lets you delete the Nth listview item.

Command: /xdid -d dialog ID N

Example: /xdid -d dcx 4 6

• -g


This command lets you set a background image to the listview control.

Command: /xdid -g dialog ID [+FLAGS] [X] [Y] (FILENAME)

Example: /xdid -g dcx 4 +n 50 50 $mircdir/images/blah.png

Parms :
[+FLAGS]Image flags
  Value :
nNormal image.
rRemove image.
tTiled Image.

[X]Normal horizontal offset (0 = left aligned, 50 = centred, 100 = right aligned).
[Y]Normal vertical offset (0 = left aligned, 50 = centred, 100 = right aligned).
(FILENAME)Filename of the image including the path.

• -i


This command lets you change the listview control colors.

Command: /xdid -i dialog ID [+FLAGS] [COLOR]

Example: /xdid -i dcx 4 +bk none

Parms :
[+FLAGS]Color flags
  Value :
bBackground color.
kBackground text color
oOutline color. (see borderselect style)
tText color.

[COLOR]RGB Color

N.B. [COLOR] can be none for the two background colors for no colors.
N.B. if [COLOR] is none for text background, the text background parameter of /xdid -a won't work anymore.

• -k


This command lets you change the check state on the Nth listview item.

Command: /xdid -k dialog ID [STATE] N

Example: /xdid -k dcx 4 2 5

Parms :
[STATE]Check State
  Value :
0No checkbox
1Unchecked
2Checked

N.B. If you are not using the checkbox style, this function can also change the state icon of an item. Be sure to insert the state icons before you use them.

• -l


This command lets you change the listview item icon.

Command: /xdid -l dialog ID [N] [NSUB] [ICON]

Example: /xdid -l dcx 4 1 2 5

Parms :
[N]Item index
[NSUB]Column index (0 for first column)
[ICON]Icon index (0 for no icon)

• -u


This command lets you trigger the group view in the listview which is off by default. (XP+)

Command: /xdid -m dialog ID [1|0]

Example: /xdid -m dcx 4 1

• -n


This command lets you change the Nth listview column width (report mode only).

Command: /xdid -n dialog ID [N] [+FLAGS] [WIDTH]

Example: /xdid -n dcx 4 2 + 100

Parms :
[+FLAGS]Header flags
  Value :
aAutosize.
hAutosize according to header text width.

[WIDTH]Header width (Value ignored if you use flags +a or +h).
N.B. if the flag +h is used for the last column, the column will be resized to the width of the control.

• -q


This command lets you add a group to the listview. (XP+)

Command: /xdid -q dialog ID [N] [+FLAGS] [GROUPID] [Group Text]

Example: /xdid -q dcx 4 1 +l 101 Group 101!

Parms :
[N]Group index order
[+FLAGS]Group flags
  Value :
cGroup text is centered.
lGroup text is left aligned.
rGroup text is right aligned.
[GROUPID]Id that identifies the group when adding items in /xdid -a.
[Group Text]Text of the group

• -r


This command lets you clear all the listview items.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -t


This command lets you set the different column header text, width and alignment. (Report style only)

Command: /xdid -t dialog ID [+FLAGS] [#ICON] [WIDTH] headtext[TAB][+FLAGS] [#ICON] [WIDTH] headtext[TAB]...

Example: /xdid -t dcx 4 +l 2 130 column 1 $chr(9) +c 1 130 column 2 $chr(9) +rb 2 130 column 3

Parms :
[+FLAGS]Item flags
  Value :
aAutosize.
bHeader icon appears right of text.
cHeader text is centered.
hAutosize according to header text width.
lHeader text is left justified.
rHeader text is right justified.

[#ICON]Icon index number from the icon list (0 for no icon).
[WIDTH]Column width in pixels.

N.B.The leftmost column is always left aligned. If you want a leftmost column aligned other than left, create a zero width left dummy column.
N.B. If you re-issue this command while the listview is populated, all the data except the first column will disappear, so you will have to repopulate the list. N.B. if the flag +h is used for the last column, the column will be resized to the width of the control.

• -u


This command makes the currently selected listview item unselected.

Command: /xdid -u dialog ID

Example: /xdid -u dcx 4

• -v


This command lets you change the listview item text.

Command: /xdid -v dialog ID [N] [NSUB] (Item Text)

Example: /xdid -v dcx 4 1 2 5

Parms :
[N]Item index
[NSUB]Column index (0 for first column)
(ItemText)Item Text (optional)

• -w


This command lets you add an icon to the listview image lists.

Command: /xdid -w dialog ID [+FLAGS] [INDEX] [FILENAME]

Example: /xdid -w dcx 4 +n 113 C:/mIRC/shell.dll

Parms :
[+FLAGS]Image list flags
  Value :
nNormal icon list.
sstate icon list.

[INDEX]Icon index in icon archive (use 0 if the file is a single icon file)
[FILENAME]Icon archive filename

• -y


This command lets you clear the listview image lists.

Command: /xdid -y dialog ID [+FLAGS]

Example: /xdid -y dcx 4 +ns

Parms :
[+FLAGS]Image list flags
  Value :
nNormal icon list.
sState icon list.

• -z


This command lets you sort the data in the listview control.

Command: /xdid -z dialog ID [+FLAGS] [COLUMN] (ALIAS)

Example: /xdid -z dcx 5 +ca 2 lvsort

Parms :
[+FLAGS]Sort flags
  Value :
aAscendant order sorting. (Default)
cCustom sorting provided by (ALIAS). Sorted values given by $1 and $2. (See Note)
dDescendant order sorting.
nNumerical sorting.
sCase-sensitive sorting. (With Alpha-numeric)
tAlpha-numeric sorting. (Default)

[COLUMN]Column to be used for sorting (1-index)
(ALIAS)Alias name to be called in the case of custom sorting. (Not needed if not custom sorting)

Note: The custom alias must return the following values for the comparison of custom values. The ascendant or descendant order is managed by the DLL with the given values returned.
$1 < $2return -1
$1 > $2return 1
$1 == $2return 0


>> $xdid() Properties


The following properties are used to retreive information in the listview control.

• text


This property lets you retreive a Nth listview item text. (Use a NSUB value of 0 for the first column item text)

Identifier: $xdid(dialog, ID, N, NSUB).text

Example: //echo -a $xdid(dcx, 4, 1, 2).text

• .seltext


This property lets you retreive the selected listview item text. (Use a NSUB value of 0 for the first column item text)

Identifier: $xdid(dialog, ID, SUBN).seltext

Example: //echo -a $xdid(dcx, 4, 0).seltext

• .sel


This property lets you retreive the item number of selected item in the listview. If the listview is multiline, then a comma-separated list of the selected lines is returned.

Identifier: $xdid(dialog, ID).sel

Example: //echo -a $xdid(dcx, 4).sel

• .selnum


This property lets you retreive the number of selected items in the listview.

Identifier: $xdid(dialog, ID).selnum

Example: //echo -a $xdid(dcx, 4).selnum

• .find


This property lets you retreive the listview item index and column index of the Nth found listview item matching the input search patern. If N is 0, returns the total number of matching items.

Identifier: $xdid(dialog, ID, {TAB}Matchtext{TAB}, T, M, N).find

Example: //echo -a $xdid(dcx, 4, $chr(9) *Item* $chr(9), W, 0, 3).find

Parms :
MatchtextString composed of wildcards or regex paterns used for the search.
TValue indicating if the search is using a regex patern or wildcard string.
  Value :
WMatchtext is a wildcard string.
RMatchtext is a regex patern.

MParameter indicating the column to search. If M is 0, searchs through all columns.
NParameter indicating to return the Nth match. If N is 0, returns the total matches found.

N.B. The {TAB} characters surrounding the matchtext are mandatory or else it won't work.

N.B. The global search including all columns goes left to right, line by line, starting at the top.

• num


This property lets you retreive the number of items of a listview.

Identifier: $xdid(dialog, ID).num

Example: //echo -a $xdid(dcx, 4).num

• state


This property lets you retreive the check item state of the Nth listview item. Returns 2: checked, 1: unchecked, 0: no check.

Identifier: $xdid(dialog, ID, N).state

Example: //echo -a $xdid(dcx, 4, 2).state

N.B. If you are not using the checkbox style, the value returned is the state icon value.

• icon


This property lets you retreive a Nth listview item icon number. (Use a NSUB value of 0 for the first column item icon)

Identifier: $xdid(dialog, ID, N, NSUB).icon

Example: //echo -a $xdid(dcx, 4, 1, 2).icon

• tbitem


This property lets you retreive top and bottom visible item indexes. (0 for no items or error)

Identifier: $xdid(dialog, ID).tbitem

Example: //echo -a $xdid(dcx, 4).tbitem

• mouseitem


This property lets you retreive the item index over which the mouse pointer is. (0 for no item where mouse lies)

Identifier: $xdid(dialog, ID).mouseitem

Example: //echo -a $xdid(dcx, 4).mouseitem

• hwidth


This property lets you retreive the Nth listview header width.

Identifier: $xdid(dialog, ID, N).hwidth

Example: //echo -a $xdid(dcx, 4, 2).hwidth

• htext


This property lets you retreive the Nth listview header text.

Identifier: $xdid(dialog, ID, N).htext

Example: //echo -a $xdid(dcx, 4, 2).htext

• hicon


This property lets you retreive the Nth listview header icon index.

Identifier: $xdid(dialog, ID, N).hicon

Example: //echo -a $xdid(dcx, 4, 2).hicon

• genabled


This property lets you retreive whether groups are enabled or not.

Identifier: $xdid(dialog, ID).genabled

Example: //echo -a $xdid(dcx, 4).genabled


>> Control Events


The following events are fired when a user does some particular actions on the listview control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   When an item is single clicked.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
dclick   When an item is double clicked
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
rclick   When an item is right clicked.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
stateclick   When a checkbox is clicked.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
labelbegin   When item text label editing begins.
 
Returnnoedit -> Return this value to block label editing.
labelend   When the user clicks elsewhere or presses RETURN while editing or clicks somewhere else but editied the text.
 
[VALUE]Edited Text label.
Returnnoedit -> Return this value to cancel label editing (for example if string is NULL or invalid to your application).
labelcancel   When the user presses ESC while editing or clicks somewhere else without editing the text.
 
hsclick   When the user clicks on a header item in the listview.
 
[VALUE]Header item index.
hdclick   When the user double clicks on a header item in the listview.
 
[VALUE]Header item index.
trackbegin   When a header item is being resized.
 
Returnnotrack -> Return this value to block the header column resizing.
 
scrollend   When the user has finished scrolling the vertical scrollbar in the listview.
 
help   Launched when you click on a control using the ? contexthelp button.
