>> Control Styles


These control styles are available when creating a web control.

Control Styles
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -g


This command is like hitting the Home button in IE.

Command: /xdid -g dialog ID

Example: /xdid -g dcx 4

• -i


This command is like hitting the Forward button in IE.

Command: /xdid -i dialog ID (Text)

Example: /xdid -i dcx 4 Check Text

• -j


This command lets you execute some javascript code.

Command: /xdid -j dialog ID [JAVASCRIPT]

Example: /xdid -j dcx 4 alert('blah')

• -k


This command is like hitting the Back button in IE.

Command: /xdid -k dialog ID (Text)

Example: /xdid -k dcx 4 Check Text

• -n


This command lets you navigate to an url.

Command: /xdid -n dialog ID [URL]

Example: /xdid -n dcx 4 http://dcx.scriptsdb.org

• -r


This command is like hitting the Refresh button in IE.

Command: /xdid -r dialog ID (Text)

Example: /xdid -r dcx 4 Check Text

• -t


This command is like hitting the Stop button in IE.

Command: /xdid -t dialog ID

Example: /xdid -t dcx 4

>> $xdid() Properties


The following properties are used to retreive information in the web control.

• url


This property lets you retreive the current loaded url.

Identifier: $xdid(dialog, ID).url

Example: //echo -a $xdid(dcx, 4).url

• ready


This property lets you retreive the ready state. If it's "$true" the control is ready for another command, else you should wait.

Identifier: $xdid(dialog, ID).ready

Example: //echo -a $xdid(dcx, 4).ready

>> Control Events


The following events are fired when a user does some particular actions on the web control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

nav_begin   When the control is about to navigate to an URL.
 
Returncancel -> Return this value to block the navigation.
 
nav_complete   When the control is finished navigating to the URL.
 
[VALUE]URL navigated to.
 
doc_complete   When the document has been downloaded.
 
[VALUE]Document URL.
 
dl_begin   When downloading is about to begin.
 
dl_progress   Downloading progress update.
 
[VALUE][POS] [MAX] Current dowload position and maximum download position.
 
dl_complete   When the downloading is complete.
 
win_open   When a new window is attempted to be opened.
 
Returncancel -> Return this value to block the new window from opening.
 
status   When the statusbar changes.
 
[VALUE]New status text.
 
title   when the title changes.
 
[VALUE]New title text.
 
forward   When the state of the forward button changes.
 
[VALUE]$true or $false indicating the enabled state of the button.
 
back   When the state of the back button changes.
 
[VALUE]$true or $false indicating the enabled state of the button.
 
help   Launched when you click on a control using the ? contexthelp button.
