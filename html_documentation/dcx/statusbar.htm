>> Introduction


This control enables you to create a status bar.

>> Control Styles


These control styles are available when creating a status bar control.

Control Styles
grip   Status bar displays a sizing grip.
tooltips   Status bar has tooltips (tooltips only work for cells that have no text or cells where the text does not fit in the cell width).
top   The statusbar is auto-positioned at top of dialog window.
noresize   Status isn't auto-resized when dialog's size changes.
noparentalign   Status isn't auto-positioned when dialog's size changes.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -k


This command lets you set the status bar background color. You can use -1 to revert to default color.

Command: /xdid -k dialog ID COLOR

Example: /xdid -k dcx 4 $rgb(255,255,255)

• -l


This command lets you set the status bar cell divisions. Every value givin is the position of the cell's right border. For example, if you have a 600 pixel width statusbar, you could use the values 200 400 -1 to get 3 200 pixel wide cells. If a cell has the value -1 for right border position, the cell will extend to the right edge of the statusbar control.

Command: /xdid -l dialog ID [POS [POS POS ...]]

Example: /xdid -l dcx 4 200 400 -1

• -t


This command lets you set the icon, text and tooltip of a statusbar cell.

Command: /xdid -t dialog ID N [+FLAGS] [#ICON] (Cell Text)[TAB]Tooltip

Example: /xdid -t dcx 4 2 +p 0 Cell Text $chr(9) Cell Text

Parms :
NStatus bar cell number to modify.
[+FLAGS]Status bar cell flags.
  Value :
pPopup style border around cell.
nNo borders around cell.

[#ICON]Icon number for the statusbar cell (0 for no icon)

N.B. Cell text is optional.
N.B. Tooltip is only displayed if cell text is absent or cell text is too long to fit in cell.

• -v


This command lets you change the text of a statusbar cell (Text is optional).

Command: /xdid -v dialog ID [N] (Text)

Example: /xdid -v dcx 4 2 New Text

• -w


This command lets you add an icon to the statusbar image list.

Command: /xdid -w dialog ID [INDEX] [FILENAME]

Example: /xdid -w dcx 4 113 C:/mIRC/shell.dll

Parms :
[INDEX]Icon index in icon archive (use 0 if the file is a single icon file)
[FILENAME]Icon archive filename

• -y


This command lets you clear the statusbar image list.

Command: /xdid -y dialog ID

Example: /xdid -y dcx 4

>> $xdid() Properties


The following properties are used to retreive information in the status bar control.

• .text


This property lets you retreive the Nth status bar cell text.

Identifier: $xdid(dialog, ID, N).text

Example: //echo -a $xdid(dcx, 4, 2).text

• .parts


This property lets you retreive the status bar cell division parts.

Identifier: $xdid(dialog, ID).parts

Example: //echo -a $xdid(dcx, 4).parts

• .tooltip


This property lets you retreive the status bar tooltip of the Nth cell.

Identifier: $xdid(dialog, ID, N).tooltip

Example: //echo -a $xdid(dcx, 4, 3).tooltip

>> Control Events


The following events are fired when a user does some particular actions on the statusbar control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   Single left click over a status bar cell.
 
[VALUE][CELL]
[CELL]Cell number where the event was triggered.
dclick   Double-click over a status bar cell.
 
[VALUE][CELL]
[CELL]Cell number where the event was triggered.
rclick   Right-click over a status bar cell.
 
[VALUE][CELL]
[CELL]Cell number where the event was triggered.
 
help   Launched when you click on a control using the ? contexthelp button.
