>> Introduction


This control enables you to create a treeview.


>> Control Styles


These control styles are available when creating a treeview control.

Control Styles
haslines   Items are linked with lines.
linesatroot   Enables lines to be connected to the root item (needs the haslines style to work).
hasbuttons   Enables ± buttons on the side of items containing child items for expand/collapse operations.
checkbox   Enables Checkbox support in the treeview.
editlabel   Makes the treeview items editable.
nohscroll   Disables horizontal scrolling in the treeview.
noscroll   Disables scrolling in the treeview.
notooltips   Disables tooltip popup on items.
showsel   Selection is always visible, even when control loses focus.
fullrow   Selection highlight is the full row of the item. Cannot be used with haslines.
singleexpand   Causes the item being selected to expand and the item being unselected to collapse upon selection in the tree view. If the user holds down the CTRL key while selecting an item, the item being unselected will not be collapsed.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add an item to the treeview.

Command: /xdid -a dialog ID N N N ... N[TAB] [+FLAGS] [#ICON] [#SICON] [#OVERLAY] [#STATE] [#INTEGRAL] [COLOR] Item Text[TAB]Tooltip Text

Example: /xdid -a dcx 4 $+(-1 2 4,$chr(9),+bc 1 1 0 0 0 $rgb(0,0,255) Item Text,$chr(9),Tooltip)

Parms :
[+FLAGS]Item flags
  Value :
bItem text is bold.
cItem text is colored ([COLOR] is the text color).
eItem is explanded.
sItem appears selected.
uItem text is underlined.

[#ICON]Icon index number from the normal icon list (0 for no icon)
[#SICON]Selected Icon index number from the normal icon list (0 for no icon)
[#OVERLAY]Icon index number from the normal icon list that was marked as an overlay icon (0 for no icon) (Not functionnal for the moment)
[#STATE]Icon index number from the state icon list (0 for no icon) (If you are using the checkbox style -> 2: checked, 1: not checked, 0: no check)
[#INTEGRAL]Multiple of the regular treeview item height (0 = normal height, 1 = twice the height, 2 = 3X height etc.)
[COLOR]Item text color of the treeview item (need to add switch +c to work). Color is in $rgb() format.

N.B. Icons must be added prior to be used in the treeview.

• -c


This command lets you select a treeview item.

Command: /xdid -c dialog ID N N N ... N

Example: /xdid -c dcx 4 1 3 5

• -d


This command lets you delete a treeview item (and it's sub-items if any).

Command: /xdid -d dialog ID N N N ... N

Example: /xdid -d dcx 4 1 3 5

• -g


This command lets you set the treeview item height (in pixels).

Command: /xdid -g dialog ID [HEIGHT]

Example: /xdid -g dcx 4 15

N.B. you can use the value -1 to reset to the default height.

• -i


This command lets you change treeview colors.

Command: /xdid -i dialog ID [+FLAGS] [COLOR]

Example: /xdid -i dcx 4 +b $rgb(0,255,0)

Parms :
[+FLAGS]Color flags
  Value :
bBackground color.
lLine color.
tText color.

[COLOR]Color in $rgb() format. Use -1 to revert to default color.

• -j


This command lets you change the icon and selected icon of a treeview item.

Command: /xdid -j dialog ID N N N ... N [TAB] [ICON] [SICON]

Example: /xdid -j dcx 4 1 2 $chr(9) 3 3

• -k


This command lets you change the check state on a treeview item.

Command: /xdid -k dialog ID [STATE] N N N ... N

Example: /xdid -k dcx 4 2 1 3 5

Parms :
[STATE]Check State
  Value :
0No checkbox
1Unchecked
2Checked

N.B. If you are not using the checkbox style, this function can also change the state icon of an item. Be sure to insert the state icons before you use them.

• -l


This command lets you change the size of the treeview icons (Default size: 16).

Command: /xdid -l dialog ID [16|24|32]

Example: /xdid -l dcx 4 24

N.B. You need to use this command before addinf icons to a treeview.

• -m


This command lets you move a treeview item and it's sub-items. The first path is the item to be moved to the second path.

Command: /xdid -m dialog ID N N N ... N{TAB}N N N ... N

Example: /xdid -m dcx 4 1 2 $chr(9) 1

• -n


This command lets you copy a treeview item and it's sub-items. The first path is the item to be copied to the second path.

Command: /xdid -n dialog ID N N N ... N{TAB}N N N ... N

Example: /xdid -n dcx 4 1 2 $chr(9) 1

• -o


This command lets you change the tooltip text of a treeview item.

Command: /xdid -o dialog ID N N N ... N [TAB] (TipText)

Example: /xdid -o dcx 4 1 2 $chr(9) Blah

• -q


This command lets you change the font styles and color on a treeview item.

Command: /xdid -q dialog ID [+FLAGS] [COLOR] N N N ... N

Example: /xdid -q dcx 4 +bcu $rgb(128,0,255) 1

Parms :
[+FLAGS]Check State
  Value :
bItem text is bold.
uItem text is underlined.
cItem text is colored ([COLOR] is the text color).

[COLOR]Item text color of the treeview item (need to add switch +c to work). Color is in $rgb() format.

N.B. Use -1 as [COLOR] value if you omit the +c switch.

• -r


This command lets you clear all the treeview items.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -t


This command lets you toggle/expand/collapse treeview items. You can use root as the path with the +a and +z switches.

Command: /xdid -t dialog ID [+FLAGS] N N N ... N

Example: /xdid -t dcx 4 +a root

Parms :
[+FLAGS]Toggle flags
  Value :
aExpands the branch and all sub-branches.
cCollapses the branch.
eExpands the branch.
pExpands partially the branch.
rCollapses and deletes the branches children.
tToggles opposite action on branch.
zCollapses the branch and all sub-branches.

• -u


This command makes the currently selected treeview item unselected.

Command: /xdid -u dialog ID

Example: /xdid -u dcx 4

• -v


This command lets you change the text of a treeview item.

Command: /xdid -v dialog ID N N N ... N [TAB] (ItemText)

Example: /xdid -v dcx 4 1 2 $chr(9) Blah

• -w


This command lets you add an icon to the treeview image lists.

Command: /xdid -w dialog ID [+FLAGS] [INDEX] [FILENAME]

Example: /xdid -w dcx 4 +n 113 C:/mIRC/shell.dll

Parms :
[+FLAGS]Image list flags
  Value :
nNormal icon list.
sState icon list.

[INDEX]Icon index in icon archive (use 0 if the file is a single icon file)
[FILENAME]Icon archive filename

• -y


This command lets you clear the treeview image lists.

Command: /xdid -y dialog ID [+FLAGS]

Example: /xdid -y dcx 4 +ns

Parms :
[+FLAGS]Image list flags
  Value :
nNormal icon list.
sState icon list.

• -z


This command lets you sort the data in the treeview control.

Command: /xdid -z dialog ID [+FLAGS] N N N ... N [TAB] (ALIAS)

Example: /xdid -z dcx 5 +tab root tvsort

Parms :
[+FLAGS]Sort flags
  Value :
aAscendant order sorting. (Default)
bSorts only the given branch.
cCustom sorting provided by (ALIAS). Sorted values given by $1 and $2. (See Note)
dDescendant order sorting.
nNumerical sorting.
rSorts recursively all sub-branches.
sCase-sensitive sorting. (With Alpha-numeric)
tAlpha-numeric sorting. (Default)

(ALIAS)Alias name to be called in the case of custom sorting. (Not needed if not custom sorting)

Note: The custom alias must return the following values for the comparison of custom values. The ascendant or descendant order is managed by the DLL with the given values returned.
$1 < $2return -1
$1 > $2return 1
$1 == $2return 0


>> $xdid() Properties


The following properties are used to retreive information in the treeview control.

• text


This property lets you retreive a treeview item text.

Identifier: $xdid(dialog, ID, ItemPath).text

Example: //echo -a $xdid(dcx, 4, 1 2).text

• .seltext


This property lets you retreive the selected treeview item text.

Identifier: $xdid(dialog, ID).seltext

Example: //echo -a $xdid(dcx, 4).seltext

• .selpath


This property lets you retreive the ItemPath that leads to the selected item in the treeview.

Identifier: $xdid(dialog, ID).selpath

Example: //echo -a $xdid(dcx, 4).selpath

• .find


This property lets you retreive a treeview ItemPath that leads to the Nth found treeview item matching the input search patern. If N is 0, returns the total number of matching items. SubPath is an optionnal parameter to start the search at a desired branch.

Identifier: $xdid(dialog, ID, {TAB}Matchtext{TAB}, T, N, SubPath).find

Example: //echo -a $xdid(dcx, 4, $chr(9) *Item* $chr(9), W, 3, 1 2).find

Parms :
MatchtextString composed of wildcards or regex paterns used for the search.
TValue indicating if the search is using a regex patern or wildcard string.
  Value :
WMatchtext is a wildcard string.
RMatchtext is a regex patern.

NParameter indicating to return the Nth match. If N is 0, returns the total matches found.
SubPathOptionnal parameter to start the search at a specific branch. (If you leave this blank, searching starts at root of treeview)

N.B. The {TAB} characters surrounding the matchtext are mandatory or else it won't work.

• num


This property lets you retreive the number of child items of a treeview item. You can use the value root as ItemPath to get the number of childs of the root item.

Identifier: $xdid(dialog, ID, ItemPath).num

Example: //echo -a $xdid(dcx, 4, 1 2).num

• state


This property lets you retreive the check item state of an item. Returns 2: checked, 1: unchecked, 0: no check.

Identifier: $xdid(dialog, ID, ItemPath).state

Example: //echo -a $xdid(dcx, 4, 1 2).state

N.B. If you are not using the checkbox style, the value returned is the state icon value.

• expand


This property lets you retreive whether a treeview item is expanded or not.

Identifier: $xdid(dialog, ID, ItemPath).expand

Example: //echo -a $xdid(dcx, 4, 1 2).expand

• icon


This property lets you retreive the treeview item icon.

Identifier: $xdid(dialog, ID, ItemPath).icon

Example: //echo -a $xdid(dcx, 4, 1 2).icon

• tooltip


This property lets you retreive the treeview item tooltip.

Identifier: $xdid(dialog, ID, ItemPath).tooltip

Example: //echo -a $xdid(dcx, 4, 1 2).tooltip

• mouseitem


This property lets you retreive the treeview ItemPath over which the mouse pointer is. (0 for no item where mouse lies)

Identifier: $xdid(dialog, ID).mouseitem

Example: //echo -a $xdid(dcx, 4).mouseitem

>> TreeView Item Path


The treeview control has an engin that parses the given item "path" data to know where to insert/delete/modify a given item in the control. The parameters you supply are a space separated numerical token list of the treeview branches that lead to the item position :

Format: N N N ... N

For example : 1 2 3

This position represents the 3rd child branch of the 2nd child branch of the first branch as shown by the selected item in the picture below:


You can also use the value -1 for any of the values to indicate the parser that you want to insert the item at the end of the branch (if used as the last path number) or to select the last branch of a given sub-path (if not used as last path number).

For example : -1 -1

This would add the item at the end of the last branch in the first column of branches.

N.B. If you specify a path that does not exist the engin will reverse to the last valid path encoutered during parsing.


>> Control Events


The following events are fired when a user does some particular actions on the treeview control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   When an item is single clicked. (N.B. You can't use $xdid().tvseltext to get item text, you need to use $xdid().tvtext with item path given)
 
[VALUE]TreeView Path leading to the item that triggered the eventd.
dclick   When an item is double clicked. (N.B. You can't use $xdid().tvseltext to get item text, you need to use $xdid().tvtext with item path given)
 
[VALUE]TreeView Path leading to the item that triggered the event.
rclick   When an item is right clicked. (N.B. You can't use $xdid().tvseltext to get item text, you need to use $xdid().tvtext with item path given)
 
[VALUE]TreeView Path leading to the item that triggered the event.
stateclick   When a checkbox is clicked.
 
[VALUE][STATE] [TreeView Path]
[STATE]New item check state (checkbox style) or state icon value.
[TreeView Path]leading to the item that triggered the event.
buttonclick   When a ± button is clicked.
 
[VALUE]TreeView Path leading to the item that had his ± button clicked.
labelbegin   When item text label editing begins.
 
Returnnoedit -> Return this value to block label editing.
labelend   When the user clicks elsewhere or presses RETURN while editing or clicks somewhere else but editied the text.
 
[VALUE]Edited Text label.
Returnnoedit -> Return this value to cancel label editing (for example if string is NULL or invalid to your application).
labelcancel   When the user presses ESC while editing or clicks somewhere else without editing the text.
 
expand   When a branch is expanded.
 
[VALUE]TreeView Path leading to the item that was expanded.
collapse   When a branch is collapsed.
 
[VALUE]TreeView Path leading to the item that was collapsed.
 
help   Launched when you click on a control using the ? contexthelp button.
