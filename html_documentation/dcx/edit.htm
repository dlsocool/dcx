>> Control Styles


These control styles are available when creating a edit control.

Control Styles
multi   Enables a multiline edit.
center   Text is centered.
right   Text is right justified.
autohs   Text scrolling is automatically scrolled horizontally without need of a horizontal scrollbar.
autovs   Text scrolling is automatically scrolled vertically without need of a vertical scrollbar.
hsbar   Displays a horizontal scrollbar.
vsbar   Displays a vertical scrollbar.
lowercase   Converts all characters to lowercase as they are typed into the edit control.
uppercase   Converts all characters to uppercase as they are typed into the edit control.
number   Allows only digits to be entered into the edit control. Note that, even with this set, it is still possible to paste non-digits into the edit control.
password   Displays an asterisk (*) for each character typed into the edit control. This style is valid only for single-line edit controls.
return   Specifies that a carriage return be inserted when the user presses the ENTER key while entering text into a multiline edit control in a dialog box.
readonly   Prevents the user from typing or editing text in the edit control.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add text to the end of the edit.

Command: /xdid -a dialog ID [TEXT]

Example: /xdid -a dcx 4 New Text

• -d


This command lets you delete the Nth line of text in the edit (Only works with multi).

Command: /xdid -d dialog ID [N]

Example: /xdid -d dcx 4 5

• -i


This command lets you insert a line at the Nth position in the edit. (If the multi style is not used, will overwrite the whole text)

Command: /xdid -i dialog ID [N] [TEXT]

Example: /xdid -i dcx 4 5 New Text

• -j


This command lets you trigger the password style of the edit control on the fly.

Command: /xdid -j dialog ID [1|0]

Example: /xdid -j dcx 4 1

• -o


This command lets you overwrite the Nth line in the edit. (If the multi style is not used, will overwrite the whole text)

Command: /xdid -o dialog ID [N] [TEXT]

Example: /xdid -o dcx 4 5 New Text

• -q


This command lets you set a char length limit on the edit.

Command: /xdid -q dialog ID [LIMIT]

Example: /xdid -q dcx 4 100

• -r


This command lets you clear the contents of the edit.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -t


This command lets you load the contents of a file directly in the edit.

Command: /xdid -t dialog ID [FILENAME]

Example: /xdid -t dcx 4 C:\mIRC\blah.txt

• -u


This command lets you save the contents of the edit directly in a file.

Command: /xdid -u dialog ID [FILENAME]

Example: /xdid -u dcx 4 C:\mIRC\blah.txt

>> $xdid() Properties


The following properties are used to retreive information in the edit control.

• text


This property lets you retreive the Nth line in the edit or the whole edit text.

Identifier: $xdid(dialog, ID, N).text

Example: //echo -a $xdid(dcx, 4, 1).text

N.B. If the multi is not used, you can use $xdid(dialog, ID).text to retrieve the edit text.

• num


This property lets you retreive the number of lines in the edit.

Identifier: $xdid(dialog, ID).num

Example: //echo -a $xdid(dcx, 4).num

N.B. Only works with the multi style.

• ispass


This property lets you retreive if the password style is set on the edit control.

Identifier: $xdid(dialog, ID).ispass

Example: //echo -a $xdid(dcx, 4).ispass


>> Control Events


The following events are fired when a user does some particular actions in the richedit. All events and data are sent to the callback alias associated to the dialog when marked.

 
edit   When the text is edited in the edit control.
 
return   When the Enter key is pressed in the edit control.
 
help   Launched when you click on a control using the ? contexthelp button.
