>> Introduction


This control enables you to create a progress bar.

>> Control Styles


These control styles are available when creating a progress bar control.

Control Styles
smooth   Progress bar is smooth (no line breaks in the bar).
vertical   Progress bar is vertical rather than horizontal.
marquee   Progress bar is an animated bar. (XP+)
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -c


This command lets you set the progress bar color.

Command: /xdid -c dialog ID RGB

Example: /xdid -c dcx 4 $rgb(255,255,255)

N.B. Only works with the notheme style.

• -i


This command lets you set the progress bar text. Text is optional and if omited, no text will be displayed. If you put the caracters %d in the text, it will be replaced by the position value of the progressbar as defined by the -j switch. Putting two %% signs together in the text produces a single % symbol in the displayed text. Default text is %d %%.

Command: /xdid -i dialog ID (Item Text)

Example: /xdid -i dcx 4 Completed % $+ d % $+ %

N.B. Remember that mIRC uses %d as a variable name, you will have to use % $+ d to insert the text in the control.

• -j


This command lets you set the progress bar numerical representation where %d is replaced in the progressbar text.

Command: /xdid -j dialog ID [MODE]

Example: /xdid -j dcx 4 p

Parms :
[MODE]Mode letter
  Value :
aAbsolute position value.
pPercentage position value (default).

• -k


This command lets you set the progress bar background color.

Command: /xdid -k dialog ID RGB

Example: /xdid -k dcx 4 $rgb(255,255,255)

N.B. Only works with the notheme style.

• -m[o|g]


This command lets you start/stop the progress bar marquee animation (-mo: start | -mg: stop ). The N value is the interval in milliseconds between each animation (default 100 ms).

Command: /xdid -m[o|g] dialog ID N

Example: /xdid -mo dcx 4 50

• -q


This command lets you set the progressbar text color. Default is black.

Command: /xdid -q dialog ID RGB

Example: /xdid -q dcx 4 $rgb(0,0,0)

• -r


This command lets you modify the progressbar range of values. (default value 0 - 100)

Command: /xdid -r dialog ID RLow RHigh

Example: /xdid -r dcx 4 0 50

• -t


This command lets you increase the progressbar by a step defined by the -u flag. When the position exceeds the maximum range value, the progress bar resets the current position so that the progress indicator starts over again from the beginning.

Command: /xdid -t dialog ID

Example: /xdid -t dcx 4

• -u


This command lets you set the progressbar step increase when using the -t switch (default value is 10).

Command: /xdid -u dialog ID N

Example: /xdid -u dcx 4 20

• -v


This command lets you set the progressbar value.

Command: /xdid -v dialog ID N

Example: /xdid -v dcx 4 50

>> $xdid() Properties


The following properties are used to retreive information in the progressbar control.

• .value


This property lets you retreive the progress bar position value.

Identifier: $xdid( dialog, ID ).value

Example: //echo -a $xdid( dcx, 4 ).value

• .range


This property lets you retreive the progress bar range values.

Identifier: $xdid( dialog, ID ).range

Example: //echo -a $xdid( dcx, 4 ).range

• .text


This property lets you retreive the progress bar text, with the current value.

Identifier: $xdid( dialog, ID ).text

Example: //echo -a $xdid( dcx, 4 ).text

>> Control Events


The following events are fired when a user does some particular actions on the progressbar control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.
sclick   When you left-click on the progressbar.
 
[VALUE][N] [MIN] [MAX]
[N]Position where mouse is when event is fired.
[MIN]Minimum range value.
[MAX]Maximum range value.
rclick   When you right-click on the progressbar.
 
[VALUE][N] [MIN] [MAX]
[N]Position where mouse is when event is fired.
[MIN]Minimum range value.
[MAX]Maximum range value.
mousebar   When the mouse is moved on the progressbar while the left mouse button is held down.
 
[VALUE][N] [MIN] [MAX]
[N]Position where mouse is when event is fired.
[MIN]Minimum range value.
[MAX]Maximum range value.
 
help   Launched when you click on a control using the ? contexthelp button.
