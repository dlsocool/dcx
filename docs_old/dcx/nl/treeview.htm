>> Introductie


Deze control stelt je in staat om een treeview te maken.


>> Control Stijlen


Deze control stijlen zijn beschikbaar wanneer je een treeview control aanmaakt .
haslines   Items zijn aanelkaar gelinked met lijnen.
linesatroot   Lijnen leiden ook naar de root item toe (heeft de haslines stijl nodig om te werken).
hasbuttons    Items met child items hebben rechts ± knoppen om ze in- of uit te klappen.
checkbox   Een aanvinkhokje kan zich zich rechts van de items bevinden.
editlabel   Maakt de tekst aanpasbaar.
nohscroll   Voorkomt dat er een horizontale scrollbar verschijnt.
noscroll   Er mag niet gescrolled worden in de treeview.
notooltips   Geen tooltips.
showsel   Selectie is altijd zichtbaar, ook als deze de focus verliest.
fullrow   Selectie highlight omvat de hele rij. Kan niet worden gebruikt in combinatie met haslines.
singleexpand    Een item kan maar uitgelklapt zijn, als een andere word uitgelkapt zal diegene die open was dichtgaan. Door Ctrl ingedrukt te houden kan de gebruiker dit tegengaan.
notheme   XP Themes heeft geen vat op deze control.

>> /xdid flags


Control commando's worden ingevoerd met behulp van het /xdid commando.

• -a


Dit commando voegt een item toe aan de treeview.

Command: /xdid -a dialog ID N N N ... N[TAB] [+FLAGS] [#ICON] [#SICON] [#OVERLAY] [#STAAT] [#INTEGRAAL] [KLEUR] Item Text[TAB]Tooltip Text

Voorbeeld: /xdid -a dcx 4 $+(-1 2 4,$chr(9),+bc 1 1 0 0 0 $rgb(0,0,255) Item Text,$chr(9),Tooltip)

Parms :
[+FLAGS]Item flags
  Value :
b Item text is vet.
u Item text is onderstreept.
c Item text is gekleurd ([KLEUR] is de text kleur).

[#ICON] icoon index nummer van de normale icoon lijst (0 voor geen icoon)
[#SICON] Geselecteerde icoon index nummer van de normale icoon lijst, weergegeven als item geselecteerd is(0 voor geen icoon)
[#OVERLAY] Icoon index nummer van de normale icoon lijst die was aangeduid als een overlay icoon (0 voor geen icoon) (Niet werkend op het moment )
[#STAAT] Icoon index nummer van de staat icoon lijst (0 voor geen icon) (als je de checkbox stijl gebruikt dan is -> 2: aangevinkt, 1: Niet aangevinkt, 0: geen checkbox ernaast)
[#INTEGRAAL] Vermedigvuldiging van de treeview item text hoogte (0 = normale hoogte, 1 = 2x de hoogte, 2 = 3de hoogte etc.)
[KLEUR] Item text kleur of the treeview item (need to add switch +c to work). kleur is in $rgb() format.

N.B. Icons moeten worden toegevoegd aan de treeview control voordat ze in items kunnen worden ingevoegd.

• -c


Dit commando laat je een treeview item selecteren.

Command: /xdid -c dialog ID N N N ... N

Voorbeeld: /xdid -c dcx 4 1 3 5

• -d


Dit commando verwijdert een treeview item (en zijn sub-items indien aanwezig)

Command: /xdid -d dialog ID N N N ... N

Voorbeeld: /xdid -d dcx 4 1 3 5

• -i


Dit commando verandert de treeview kleuren.

Command: /xdid -i dialog ID [+FLAGS] [KLEUR]

Voorbeeld: /xdid -i dcx 4 +b $rgb(0,255,0)

Parms :
[+FLAGS]kleur flags
  Value :
b Achtergrond kleur .
l Lijn kleur .
t Text kleur.

[KLEUR] kleur is in $rgb() formaat. gebruik -1 om naar de standaard kleur terug te schakelen.

• -k


Dit commando veranderd de staat van de check van een treeview item.

Command: /xdid -k dialog ID [STATE] N N N ... N

Voorbeeld: /xdid -k dcx 4 2 1 3 5

Parms :
[STATE]Check State
  Value :
0 Geen checkbox
1 Niet aangevinkt
2 Aangevinkt

N.B. als je de checkbox stijl niet gebruikt dan gebruikt dit commando de staat icoontjes. Deze moeten worden toegevoegd voordat deze kunnen worden aangeroepen.

• -m


Dit commando verplaats treeview items. De eerste locatie is degene die word verplaatst naar de 2de locatie.

Command: /xdid -m dialog ID N N N ... N{TAB}N N N ... N

Voorbeeld: /xdid -m dcx 4 1 2 $chr(9) 1

• -n


Dit commando kopieerd treeview items. De eerste locatie is degene die word gekopieerd naar de 2de locatie.

Command: /xdid -n dialog ID N N N ... N{TAB}N N N ... N

Voorbeeld: /xdid -n dcx 4 1 2 $chr(9) 1

• -q


Dit commando veranderd het lettertype en de kleuren van een treeview item.

Command: /xdid -q dialog ID [+FLAGS] [KLEUR] N N N ... N

Voorbeeld: /xdid -q dcx 4 +bcu $rgb(128,0,255) 1

Parms :
[+FLAGS]Check State
  Value :
b Item text is vet.
u Item text is onderstreept.
c Item text is gekleurd ([KLEUR] is de text kleur).

[kleur] Item text kleur van de treeview item (heeft +c nodig om te werken). kleur is in $rgb() formaat.

N.B. gebruik -1 als [kleur] waarde als je de +c switch achterwege laat.

• -r

Dit commando verwijdert alle treeview items.

Command: /xdid -r dialog ID

Voorbeeld: /xdid -r dcx 4

• -t


Dit commando zorgt voor het toggle/uitklappen/inklappen van treeview items. Je kan root gebruiken als de path met de +a en +z switches.

Command: /xdid -t dialog ID [+FLAGS] N N N ... N

Voorbeeld: /xdid -t dcx 4 +a root

Parms :
[+FLAGS]Toggle flags
  Value :
a Klapt branch en alle sub-branches uit.
c Klapt the branch in.
e Klapt the branch uit.
p Klapt de branch gedeeltelijk uit.
r Klapt de branches zijn children in en verwijdert deze.
t Toggled tegenovergestelde actie op de branch.
z Klapt branch en alle sub-branches in

• -u


Dit commando unselecteerd de op dat moment geselecteerde item.

Command: /xdid -u dialog ID

Voorbeeld: /xdid -u dcx 4

• -v


Dit commando veranderd de text van een treeview item.

Command: /xdid -v dialog ID N N N ... N [TAB] (ItemText)

Voorbeeld: /xdid -v dcx 4 1 2 $chr(9) Blah

• -w


Dit commando laat je icoontjes toevoegen aan de treeview plaatjes lijst.

Command: /xdid -w dialog ID [+FLAGS] [INDEX] [BESTANDSNAAM]

Voorbeeld: /xdid -w dcx 4 +n 113 C:/mIRC/shell.dll

Parms :
[+FLAGS] Plaatjes lijst flags
  Value :
n Normale icoon lijst.
s Staat icoon lijst.

[INDEX] Icoon index in icoon archief (gebruik 0 als bestandsnaam maar uit 1 icoontje bestaat b.v: een .ico bestand)
[BESTANDSNAAM] Icoon archief's Bestandsnaam

• -y


Dit commando maakt de plaatjes lijst helemaal leeg.

Command: /xdid -y dialog ID [+FLAGS]

Voorbeeld: /xdid -y dcx 4 +ns

Parms :
[+FLAGS]Image list flags
  Value :
n Normale icoon lijst
s Staat icoon lijst.

• -z


This command lets you sort the data in the treeview control. Dit commando sorteert de data in de treeview control

Command: /xdid -z dialog ID [+FLAGS] N N N ... N [TAB] (ALIAS)

Voorbeeld: /xdid -z dcx 5 +tab root tvsort

Parms :
[+FLAGS]Sort flags
  Value :
a Sorteer van klein naar groot. (Standaard)
b Sorteert alleen de opgegeven branch.
c Eigenhandig sorteren doormiddel van een (ALIAS). Soorteert doormiddel van $1 en $2. (Zie Opmerking)
dDescendant order sorting.
nNumerical sorting.
rSorts recursively all sub-branches.
sCase-sensitive sorting. (With Alpha-numeric)
tAlpha-numeric sorting. (Default)

(ALIAS) Naam van de alias die voor het sorteren zorgt. (Niet nodig als +c niet gegeven is)

Opmerking: De alias moet de volgende waardes retourneren voor de vergelijking van de input. De volgorde word door de DLL gedaan.
$1 < $2return -1
$1 > $2return 1
$1 == $2return 0


>> $xdid() Properties

De volgende properties worden gebruikt om informatie van de treeview te krijgen.

• text


Deze property retourneert de tekst van een item.

Identifier: $xdid(dialog, ID, ItemPath).text

Voorbeeld: //echo -a $xdid(dcx, 4, 1 2).text

• .seltext


Deze property retourneert welk item geselecteerd is in de treeview.

Identifier: $xdid(dialog, ID).seltext

Voorbeeld: //echo -a $xdid(dcx, 4).seltext

• .selpath


Deze property retourneert de ItemPath die leid naar de item die geselecteerd is in de treeview.

Identifier: $xdid(dialog, ID).selpath

Voorbeeld: //echo -a $xdid(dcx, 4).selpath

• .find


Deze property retourneert de ItemPath die leid naar de Ne gevonden treeview item die de input string matched. Als N 0 is, dan word het totaal aantal gevonden items geretourneert . SubPath is een optionele parameter die de zoekopdract laat beginnen bij de opgegeven path.

Identifier: $xdid(dialog, ID, {TAB}Matchtext{TAB}, T, N, SubPath).find

Voorbeeld: //echo -a $xdid(dcx, 4, $chr(9) *Item* $chr(9), W, 3, 1 2).find

Parms :
Matchtext Wildcard of regex text voor de zoekopdracht
T Waarde die aanduidt of het een regex of een wildcard zoekopdracht is.
  Value :
W Matchtext is een wildcard string.
R Matchtext is een regex patroon.

N Parameter om de Ne gevonden item te retourneren. Als N 0 is, retourneert het het aantal gevonden items.
SubPath SubPath is een optionele parameter die de zoekopdract laat beginnen bij de opgegeven path. Laat dit blanco om het bij de root te laten beginnen

N.B. De {TAB} karakters die de matchtext omgeven zijn verplicht, anders werkt deze niet.

• num


Deze property retourneert het aantal "child items" van een treeview item. Je kan root als ItemPath gebruiken om het total aantal child items te krijgen..

Identifier: $xdid(dialog, ID, ItemPath).num

Voorbeeld: //echo -a $xdid(dcx, 4, 1 2).num

• state


Deze property retourneert de staat van de check bij een item. 2: aangevinkt, 1: niet aangevinkt, 0: geen aanvink hokje .

Identifier: $xdid(dialog, ID, ItemPath).state

Voorbeeld: //echo -a $xdid(dcx, 4, 1 2).state

N.B. Als je checkbox stijl niet gebruikt, dan retourneert deze property de staat icon.

• expand


Deze treevew property retourneert of een item uitgeklapt is ja of te nee.

Identifier: $xdid(dialog, ID, ItemPath).expand

Voorbeeld: //echo -a $xdid(dcx, 4, 1 2).expand

>> TreeView Item Path


De treeview control heeft een mothode/parser die de "path" data omzet om te weten waar iets word toegevoegd/verwijderd/veranderd in een gegeven item in de control. De parameters die je geeft zijn een spatie gescheiden numerieke teken lijst van de treeview takken die leiden naar een item's positie:

Format: N N N ... N

For example : 1 2 3

Deze postie geeft de 3e child branch van 2e child branch van de eerste branch weer zoals getoond in de geselecteerde item in het plaatje hieronder:

Je kan ook de waarde -1 gebruiken voor elk van de numerieke tekens om aan tegeven aan de methode/parser dat iets aan het einde wilt toevoegen ( als deze gebruikt word als laatste nummer) of om de laatste tak te gebruiken van een gegeven subpath (als het niet gebruikt word als laatste nummer)


Een voorbeeld: -1 -1
Dit voegt een item toe aan het einde van de laatste tak in de eerste kolom


N.B. Als je een niet bestaande path opgeeft dan gebruikt DCX de laatst tegengekomen path die het vond tijdens het zoeken ernaar.


>> Control Events

De volgende gebeurtenissen worden getriggered als een een gebruiker een bepaalde aktie onderneemt in de treeview. All gebeurtenissen en data worden naar de callback alias gestuurd die is opgegeven toen de dialog gemarkeerd werd.
sclick    Wanneer er een enkele klik is op een item. (N.B. Je kan$xdid().tvseltext niet gebruiken om een item text te krijgen, je moet $xdid().tvtext met de item path geven)
 
[VALUE] TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

dclick   Wanneer er een dubbele klik is op een item. (N.B. Je kan$xdid().tvseltext niet gebruiken om een item text te krijgen, je moet $xdid().tvtext met de item path geven)
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

rclick   Wanneer er een rechtermuis klik is op een item.(N.B. Je kan$xdid().tvseltext niet gebruiken om een item text te krijgen, je moet $xdid().tvtext met de item path geven)
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

stateclick   Wanneer een checkbox word aangeklikt.
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

buttonclick   Wanneer een ± knop wordt aangelklikt.
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

labelbegin   Wanneer er begonnen word met het bewerken van een item's text.
 
Return noedit -> Return deze waarde om het bewerken van de text tegen te gaan.

labelend   Wanneer er ENTER word ingetoets of als er egens buiten de selectie word geklikt tijdens het bewerken van een item's text.
 
[VALUE] Bewerkte Text label.
Returnnoedit -> Return deze waarde om het bewerken van de text tegen te gaan.

labelcancel   Wanneer er ESC word ingetoets of als er egens buiten de selectie word geklikt tijdens het bewerken van een item's text.
 
expand   Wanneer een tak word uitgeklapt.
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.

collapse   Wanneer een tak word ingeklapt.
 
[VALUE]TreeView Path Leidt naar de item waar deze gebeurtenis plaats vond.
 
help   Gelanceerd als u de control aanklikt met behulp van de ? contexthelp knop.
