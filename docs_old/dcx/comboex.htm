>> Introduction


This control enables you to create a comboex control which is a combo box extended to support images.


>> Control Styles


These control styles are available when creating a comboex.

Control Styles
simple   Combo that is a list with an edit box
dropdown   Simple dropdown combo.
dropedit   Dropdown combo with an editbox.
 
General Styles
tabstop   Specifies a control that can receive the keyboard focus when the user presses the TAB key. Pressing the TAB key changes the keyboard focus to the next control with the tabstop style.
group   Specifies the first control of a group of controls. The group consists of this first control and all controls defined after it, up to the next control with the group style. The first control in each group usually has the tabstop style so that the user can move from group to group. The user can subsequently change the keyboard focus from one control in the group to the next control in the group by using the direction keys.
disabled   The control is created disabled.
notheme   Removes XP Themes on controls.

>> /xdid flags


Control commands are input to the control with the /xdid command.

• -a


This command lets you add an item to the comboex.

Command: /xdid -a dialog ID [N] [INDENT] [#ICON] [#SELECTED] [#OVERLAY] [Item Text]

Example: /xdid -a dcx 4 3 0 1 2 0 scriptsdb.org

Parms :
[N]Position where the comboex item will be added (Use 0 to insert at the end).
[INDENT]Number of indent widths from the left border (every unit is a 10 pixel indent).
[#ICON]Icon displayed when item is not selected (Use 0 for no icon).
[#SELECTED]Icon displayed when item is selected (Use 0 for no icon).
[#OVERLAY]Overlay icon of the item (not functionnal yet)

N.B. You can use 0 for the N value to insert the item at the end of the comboex.

• -c


This command lets you select the Nth comboex item.

Command: /xdid -c dialog ID N

Example: /xdid -c dcx 4 5

• -d


This command lets you delete the Nth comboex item.

Command: /xdid -d dialog ID N

Example: /xdid -d dcx 4 6

• -r


This command lets you clear all the comboex items.

Command: /xdid -r dialog ID

Example: /xdid -r dcx 4

• -u


This command makes the currently selected comboex item unselected.

Command: /xdid -u dialog ID

Example: /xdid -u dcx 4

• -w


This command lets you add an icon to the comboex image list.

Command: /xdid -w dialog ID [INDEX] [FILENAME]

Example: /xdid -w dcx 4 113 C:/mIRC/shell.dll

Parms :
[INDEX]Icon index in icon archive (use 0 if the file is a single icon file)
[FILENAME]Icon archive filename

• -y


This command lets you clear the comboex image list.

Command: /xdid -y dialog ID

Example: /xdid -y dcx 4


>> $xdid() Properties


The following properties are used to retreive information from the comboex control.

• .text


This property lets you retreive the Nth comboex item text.

Identifier: $xdid(dialog, ID, N).text

Example: //echo -a $xdid(dcx, 4, 1).text

N.B. You can use a value of 0 for N to retrieve the text of the editbox. This only works with the dropedit style.

• .sel


This property lets you retreive the comboex selected item number.

Identifier: $xdid(dialog, ID).sel

Example: //echo -a $xdid(dialog, ID).sel

• .seltext


This property lets you retreive selected comboex item text.

Identifier: $xdid(dialog, ID).seltext

Example: //echo -a $xdid(dcx, 4).seltext

• .num


This property lets you retreive the total number of comboex items.

Identifier: $xdid(dialog, ID).num

Example: //echo -a $xdid(dcx, 4).num

• .find


This property lets you retreive the comboex item index of the Nth found comboex item matching the input search patern. If N is 0, returns the total number of matching items.

Identifier: $xdid(dialog, ID, {TAB}Matchtext{TAB}, T, N).find

Example: //echo -a $xdid(dcx, 4, $chr(9) *Item* $chr(9), W, 3).find

Parms :
MatchtextString composed of wildcards or regex paterns used for the search.
TValue indicating if the search is using a regex patern or wildcard string.
  Value :
WMatchtext is a wildcard string.
RMatchtext is a regex patern.

NParameter indicating to return the Nth match. If N is 0, returns the total matches found.

N.B. The {TAB} characters surrounding the matchtext are mandatory or else it won't work.

>> Control Events


The following events are fired when a user does some particular actions on the comboex control. All events and data are sent to the callback alias associated to the dialog when marked. Remember that the events over a child control will be fired according to that child control.

sclick   When an item is selected in the comboex.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
 
dclick   When an item is double-clicked in a simple comboex.
 
[VALUE][ITEM]
[ITEM]Item number where the event was triggered.
 
edit   When the text is edited in a comboex with editbox.
 
return   When the Enter key is pressed in the editbox of a comboex.
 
dropfile   Triggers when a file is dropped on the control.
 
help   Launched when you click on a control using the ? contexthelp button.
