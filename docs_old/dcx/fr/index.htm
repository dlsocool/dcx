>> Introduction


Cette DLL vous permet d'ajouter de nouveaux contrôles aux dialogs de mIRC en même temps que les changements d'autres caractéristiques comme le style visuel, la position, etc.

La DLL crée les contrôles directement sur le dialog de mIRC et vous utilisez les fonctions de clônage définis par /xdid et $xdid qui se comportent à l'identique de /did et $did pour interagir avec les contrôles.

La gestion de l'évènement dans la DLL est supportée par un alias de rappel fourni par l'utilisateur quad le dialog est marqué avant l'ajout de nouveaux contrôles DCX. L'alias de rappel est appelé depuis la DLL pour demander des informations (si vous voulez modifier ainsi le comportement par défaut le type de "rappel") et fournira à l'utilisateur avec les évènements et paramètres de ces évènements.

Chaque fonction dans cette DLL retourne une valeur si elle est utilisée avec l'identificateur $dll. D_OK [fonction]: [message] si succès, D_ERROR [fonction]: [message] autrement.


>> Contrôles DCX supportés


Les contrôles DCX supportés sont des contrôles étendus implantés directement sur le dialog de mIRC. Ceux-ci sont utilisés pour étendre les capacités des contrôles existants dans les dialogs de mIRC lesquels sont franchement très limités en utilisation pour la conception de GUI.

N.B. Notez SVP que certaines caractéristiques de certains contrôles DCX sont disponibles uniquement sur les machines ayant Windows XP+. Ces commandes ont un symbole (XP+) à côté de leur listage.

>> Commandes générales


Ces commandes sont des commandes DCX générales et ne sont aucunement orientées sur un contrôle DCX particulier.

• Mark


Cette commande marque le dialog pour les futures commandes.

Syntaxe : [NOM] [ALIAS DE RAPPEL]

Paramètres :
[NOM]Dialog window name.
[ALIAS DE RAPPEL]Nom de l'alias de rappel de l'évènement.

N.B. Cette commande doit être appelée avant toute autre commande ou rien ne fonctionnera.

• Version


Retourne l'information sur la version de la DLL.


>> Commande /xdialog


La commande /xdialog est utilisée pour modifier le dialog marqué DCX.

Format : /xdialog [-SWITCH] [NOM] (EXTRA)

[-SWITCH]Drapeaux de paramétrage qui déterminent l'action à accomplir sur le dialog.
[NOM]Nom du dialog.
(EXTRA)Paramètres supplémentaires en fonction de la commande.

• -a


Cette commande vous permet d'animer une ouverture/fermeture de dialog.

Commande : /xdialog -a dialog ID [+DRAPEAUX] [DELAI]

Exemple : /xdialog -a dcx +ab 1000

Paramètres :
[+DRAPEAUX]Drapeaux du style d'animation.
  Valeur :
aMontre la fenêtre.
bEffet de fondu Alpha-Blending.
cFait que la fenêtre s'effondre sur elle-même si +h est utilisé ou s'agrandit si +a est utilisé. Les différents drapeaux de direction n'ont aucun effet.
hCache la fenêtre.
oAnime la fenêtre de la gauche vers la droite. Ce drapeau peut être utilisé avec une animation de roulement ou de déplacement sur le côté. Il est ignoré si utilisé avec +c ou +a.
nAnime la fenêtre de la droite vers la gauche. Ce drapeau peut être utilisé avec une animation de roulement ou de déplacement sur le côté. Il est ignoré si utilisé avec +c ou +a.
sUtilise l'animation de déplacement sur le côté. Il est ignoré si utilisé avec +c.
vAnime la fenêtre du haut vers le bas. Ce drapeau peut être utilisé avec une animation de roulement ou de déplacement de côté. Il est ignoré si utilisé avec +c ou +a.
uAnime la fenêtre du bas vers le haut. Ce drapeau peut être utilisé avec une animation de roulement ou de déplacement de côté. Il est ignoré si utilisé avec +c ou +a.

[DELAI]Délai en ms de l'animation. (Valeur système typique est 200)

N.B. L'effet par défaut st une animation de roulement.

• -b


Cette commande modifie les attributs de bord et de style sur un dialog.

Commande : /xdialog -b dialog [+DRAPEAUX]

Exemple : /xdialog -b dcx +tyszh

Paramètres :
[+DRAPEAUX]Drapeaux de bord/style de fenêtre.
  Valeur :
bBord normal.
cBord fin de client.
dBord forme du dialog.
fBord format modal du dialog. (Similaire à +d)
hBouton aide contextuelle ?. (Nécessite +y et ne peut être utilisé avec +m ou +n)
mBouton maximiser.
nBouton minimiser.
oFenêtre outil.
sBord fin statique.
tBarre de titre.
wBord fin de fenêtre.
ySysmenu (le petit bouton X)
zBord redimmensionnable.

• -c


Cette commande crée un contrôle DCX sur votre dialog.

Commande : /xdialog -c dialog [ID] [CONTROLE] [X] [Y] [W] [H] (OPTIONS)

Exemple : /xdialog -c dcx 2 pbar 140 30 100 20 smooth notheme

Paramètres :
[ID]ID du contrôle du dialog qui servira d'hôte au contrôle DCX.
[CONTROLE]DCX Control type wanted.
  Value :
pbarCrée un contrôle Barre de Progression (Progressbar).
treeviewCrée un contrôle Vue en Arbre (Treeview).
toolbarCrée un contrôle Barre d'outils (Toolbar).
statusbarCrée un contrôle Barre de Statut (Statusbar).
comboexCrée un contrôle Boite Combo Etendue (ComboEx).
listviewCrée un contrôle Vue en Liste (Listview).
trackbarCrée un contrôle Barre Graduée (Trackbar).
rebarCrée un contrôle Rebar.
colorcomboCrée un contrôle ColorCombo.
buttonCrée un contrôle Bouton (Button).
 
dividerCrée un contrôle Diviseur (Divider).
panelCrée un contrôle Panel.
tabCrée un contrôle Tab.
 
windowDocke une @fenêtre de mIRC. Le premier paramètre dans (OPTIONS) doit être le nom de la @fenêtre.
dialogDocke un dialog de mIRC. Le premier paramètre dans (OPTIONS) doit être le nom du dialog.

[X]Position horizontale du contrôle.
[Y]Position verticale du contrôle.
[W]Largeur du contrôle.
[H]Hauteur du contrôle.
(OPTIONS)Styles optionnels et options disponibles sur chaque contrôle.

• -d


Cette commande efface un contrôle DCX dans le dialog.

Commande : /xdialog -d dialog ID

Exemple : /xdialog -d dcx 4

• -h


Cette commande cache le dialog.

Commande : /xdialog -h dialog

Exemple : /xdialog -h dcx

• -l


Cette commande permet d'ajouter des règles d'algorithme d'agencement de cellule au contrôles de votre dialog pour un redimmensionnement automatique des contrôles enfant.

Commande : /xdialog -l dialog [COMMANDE] [CHEMIN] [TAB] [+DRAPEAUX] [ID] [POIDS] [W] [H]

Exemple : /xdialog -l dcx root $chr(9) +pv 0 1 0 0

Exemple : /xdialog -l dcx root 1 2 $chr(9) +fi 7 1 200 300

Exemple : /xdialog -l dcx space 1 $chr(9) + 5 5 5 5

Paramètres :
[COMMANDE]Commande de conception.
  Valeur :
rootRègle la cellule racine.
cellAjoute uen cellule en tant qu'enfant d'une autre cellule.
spaceRègle le remplissage d'espace autour de la cellule.
updateMet à jour l'agencement de cellule (doit être utilisé après avoir ajouté des règles ou changé les réglages de replissage pour rafraichir l'affichage).

[CHEMIN]Chemin de la cellule à partir de la cellule racine (peut être root pour pointer sur l'élément racine). Voir l'explication de l'algorithme d'agencement de cellule pour des exemples concrets.
[+DRAPEAUX]Type de contrôle DCX voulu.
  Valeur :
fCrée une cellule fixée.
hCrée une cellule à largeur fixe si utilisé avec f ou un carreau de cellule horizontal si utilisé avec p.
iIndique que l'[ID] du contrôle fourni est valide et est le contrôle enfant lorsqu'il est utilisé avec f ou l.
lCrée une cellule remplie.
pCrée une cellule panneau.
vCrée une cellule à hauteur fixe si utilisée avec f ou une cellule panneau verticale si utilisé avec p.
wLorsqu'il est utilisé, il signifie que les [W] et [H] fournis représentent la largeur de cellule fixée dans la direction applicable (si la cellule fixée est fixée en largeur, hauteur ou les deux).

[ID]ID du contrôle (utilisé avec des cellules fixées ou remplies - utiliser 0 si aucun contrôle ne doit être connecté à la cellule).
[POIDS]Poids de la cellule enfant. (utilisé lors de l'ajout d'une cellule enfant à une cellule panneau)
[W]Largeur fixée d'un contrôle (utilisé avec une cellule fixée).
[H]Hauteur fixée du contrôle. (utilisé avec la cellule fixée)
N.B. Si vous utilisez la commande update dans l'évènement init d'un dialog, vous aurez à utiliser .timer 0 1 xdialog -l nom update lorsque le problème avec XP et les thèmes désactivés empêchent les fenêtres d'apparaitre. Le timer corrige ce problème.

>> Identificateur $xdialog


L'identificateur $xdialog est un alias mIRC donné qui communique avec la DLL DCX pour extraire l'information des dialogs DCX.

Format : $xdialog( NOM, EXTRA ).prop

[NOM]Nom du dialog.
[EXTRA]Paramètres optionnels nécessaires à la propriété.
propPropriété récupérant les données.

Valeur de retour : Retournera les informations nécessaires si elles existent sinon elle retourne la valeur $null.

• isid


Cette propriété indique si l'ID donnée existe dans le dialog.

Identificateur : $xdialog(dialog, ID).isid

Exemple : //echo -a $xdialog(dcx, 4).isid

• nextid


Cette propriété indique le prochain ID libre dans le dialog.

Identificateur : $xdialog(dialog).nextid

Exemple : //echo -a $xdialog(dcx).nextid


>> Evènements de l'alias de rappel


Chaque contrôle DCX lance des évènements à chaque action de l'utilisateur sur le contrôle. Ces évènements sont envoyés à un alias passé en tant que paramètre lorsque le dialog est Marqué pour recevoir en entrée de la DLL DCX et y ajouter des contrôles.

Pourquoi les évènements ne sont pas envoyés comme des évènements *:DIALOG: normaux ?

Pour une chose, les évènements de mIRC sont linéaires et statiques de la façon que vous ne pouvez retourner aucune valeur au gestionnaire d'évènement pour modifier le comportement par défaut de l'évènement. En utilisant le support de rappel avec un alias permet ce type d'interaction.

Format : [NOM] [EVENEMENT] [ID] [VALEUR]

[NOM]Nom du dialog que vous avez donné lors de la commande Mark. (paramètre $1)
[EVENEMENT]Nom de l'évènement (voir chaque contrôle pour les évènements supportés). Par exemple, sclick, dclick, ... (paramètre $2)
[ID]Dialog ID du contrôle qui a lancé l'évènement. (paramètre $3)
[VALEUR]Valeur supplémentaire du contrôle de base passée dans l'alias de rappel (généralement $null sauf si spécifié autrement dans chaque évènement du contrôle) (paramètres $4-)

Valeur de retour : Certains évènements peuvent retourner une valeur spéciale qui interagira avec la DLL modifiant le comportement par défaut de l'évènement. Les évènements qui supportent une valeur de retour auront une description de ce que la valeur de retour a lors du retour : Retour : valeur

Un court exemple pour illustrer le concept : Vous avez un contrôle vue en liste en style rapport avec des entêtes et l'utilisateur déplace le séparateur de l'entête pour les redimmensionner. Dans un gestionnaire d'évènement statique linéaire normal, vous ne pouvez pas arrêter à la volée le redimmensionnement des entêtes, mais avec le support de rappel vous pouvez, parce que lorsque vous recevez l'évènement sous la forme d'un message trackbegin vous ne retournerez rien si vous voulez laisser se faire le déplacement ou vous pouvez retourner la valeur notrack et la DLL modifiera alors son comportement pour bloquer le redimmensionnement de l'entête.


>> Commande /xdid


La commande /xdid est utilisée pour ajouter/modifier/retirer les contenus d'un contrôle DCX comme d'autres caractéristiques générales (Plus tard, j'ajouterai une fonctionnalité pour certains contrôles mIRC.).

Format : /xdid [-SWITCH] [NOM] [ID] (EXTRA)

[-SWITCH]Drapeaux de paramètre qui déterminent l'action à accomplir sur le contrôle. Voir le contrôle que vous cherchez pour cibler l'information sur ces drapeaux
[NOM]Name of the dialog window.
[ID]Numéro ID du contrôle où vous voulez récupérer l'information.
(EXTRA)Paramètres supplémentaires selon le contrôle passés pour obtenir un contrôle sur les informations.

• -b


Cette commande désactive le contrôle.

Commande : /xdid -b dialog ID

Exemple : /xdid -b dcx 4

• -e


Cette commande valide le contrôle.

Commande : /xdid -e dialog ID

Exemple : /xdid -e dcx 4

• -f


Cette commande change la fonte du contrôle.

Commande : /xdid -f dialog ID [+DRAPEAUX] [TAILLE] [NOMFONTE]

Exemple : /xdid -f dcx 4 +abs 10 Tahoma

Paramètres :
[+FLAGS]Drapeaux du style de fonte.
  Valeur :
aLa fonte est anti-aliasée.
bLa fonte est grasse.
dFonte par défaut.([TAILLE] et [NOMFONTE] peuvent être omis)
iLa fonte est en italique.
sLa fonte est barrée.
uLa fonte est soulignée.

[TAILLE]Taille de la fonte en points.
[NOMFONTE]Nom de la fonte. Ex: Tahoma, Courier New, Arial, ....

N.B. Les noms de la fonte sont limités à 31 caractères (définition système).

• -h


Cette commande cache le contrôle.

Commande : /xdid -h dialog ID

Exemple : /xdid -h dcx 4

• -p


Cette commande déplace/redimmensionne le contrôle.

Commande : /xdid -p dialog ID [X] [Y] [W] [H]

Exemple : /xdid -p dcx 4 10 10 200 120

[X]Position horizontale du contrôle.
[Y]Position verticale du contrôle.
[W]Largeur du contrôle.
[H]Hauteur du contrôle.

• -s


Cette commande montre le contrôle.

Commande : /xdid -s dialog ID

Exemple : /xdid -s dcx 4

• -x


Cette commande modifie les styles de bordure du contrôle.

Commande : /xdid -x dialog ID [+FLAGS]

Exemple : /xdid -x dcx 4 +w

Paramètres :
[+DRAPEAUX]Drapeaux de style de bordure.
  Valeur :
bBord normal.
cBord Clientedge.
dBord de structure de dialog.
fBord de structure modale du dialog. (Similaire à +d)
sBord Staticedge.
wBord Windowedge.

• -M


Cette commande permet d'ajouter une information supplémentaire au contrôle qui peut être récupérée plus tard.

Commande : /xdid -M dialog ID [INFO]

Exemple : /xdid -M dcx 4 Mark info here

>> Identificateur $xdid


L'identificateur $xdid est un alias mIRC donné qui communique avec la DLL DCX pour extraire l'information dans les contrôles DCX. (Plus tard, j'ajouterai des fonctionnalités pour certains contrôles mIRC.)

Format : $xdid( NOM, ID, EXTRA ).prop

[NOM]Nom du dialog.
[ID]Numéro ID du contrôle où< vous voulez récupérer l'information.
[EXTRA]Paramètres supplémentaires selon le contrôle passés pour obtenir un contrôle sur les données.
propPropriété récupérant les informations selon le contrôle.

Valeur de retour : Retournera l'information désirée si elle existe sinon retourne la valeur $null.

• enabled


Cette propriété dit si un contrôle est activé (enabled) ou désactivé (disabled).

Identificateur : $xdid(dialog, ID).enabled

Exemple : //echo -a $xdid(dcx, 4).enabled

• visible


Cette propriété dit si un contrôle est visible ou caché.

Identificateur : $xdid(dialog, ID).visible

Exemple : //echo -a $xdid(dcx, 4).visible

• pos


Cette propriété récupère la position et la taille du contrôle au format [X] [Y] [W] [H].

Identificateur : $xdid(dialog, ID).pos

Exemple : //echo -a $xdid(dcx, 4).pos

• mark


Cette propriété récupère l'info supplémentaire donnée à un contrôle avec /xdid -M.

Identificateur : $xdid(dialog, ID).mark

Exemple : //echo -a $xdid(dcx, 4).mark

>> Evènements de dialog


Ces évènements sont renvoyés lorsque des évènements arrivent dans le dialog lui-même.

close   Quand le bouton X est cliqué sur le dialog.
 
Retournoclose -> Retourne cette valeur pour bloquer la fermeture de la fenêtre.
 
beginsize   Lorsque le dialog est sur le point d'être redimmensionné.
 
Retournosize -> Retourne cette valeur pour bloquer le redimmensionnement de la fenêtre.
 
sizing   Lorsque le dialog est en cours de redimmensionnement.
 
endsize   Lorsque le dialog a fini d'être redimmensionné.
 
beginmove   Lorsque le dialog est sur le point d'être déplacé.
 
retournomove -> Retourne cette valeur pour bloquer le déplacement de la fenêtre.
 
moving   Lorsque le dialog est en cours de déplacement.
 
endmove   Lorsque le dialog a fini d'être déplacé.
 
min   Lorsque le dialog est sur le point d'être minimisé.
 
Retourstop -> Retourne cette valeur pour bloquer la minimisation de la fenêtre.
 
max   Lorsque le dialog est sur le point d'être maximisé.
 
Retourstop -> Retourne cette valeur pour bloquer la maximisation de la fenêtre.
 
restore   Lorsque le dialog est sur le point d'être restauré.
